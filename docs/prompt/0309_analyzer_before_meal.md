'식사기록'에 현재 '분석하기' 버튼만 있잖아.
이 '분석하기' 버튼은 '밥 먹은 후'로 변경해줘.
그리고 '밥 먹기 전' 버튼을 왼쪽에 만들어줘.

'밥 먹은 후' 버튼은 말 그대로 이미 섭취한 식단을 기록하고 분석하는 기능이야.
하지만 '밥 먹기 전' 버튼은 섭취 전 "지금 이 것을 먹어도 되는지 조언을 구하는 기능" 이야.
(즉, db에 저장할 필요는 없음)

피드백은 팝업으로 크게 띄워주면 될 것 같아.

// 핵심 로직
"기존에 작성한 app/analyzer.py와 app/analyzer_picture.py의 분석 로직 뒤에 실행될 'Safety Guard' 기능을 app/safety_guard.py로 만들어줘.

핵심 요구사항:

    데이터 소스: user_profile.json에서 사용자의 지병(예: 당뇨, 고혈압), 복용 중인 약물, 주의 성분 정보를 읽어와야 함.

    위험 성분 경고: 분석된 음식의 당 함량이 높으면 당뇨 환자에게 '혈당 스파이크' 경고를, 나트륨이 높으면 고혈압 환자에게 '혈압 상승 주의'와 함께 '양을 절반으로 줄이라'는 구체적 가이드를 줄 것.

    약물-음식 상호작용: 사용자가 복용 중인 약물과 상극인 음식을 체크 (예: 고혈압 약 - 자몽, 와파린 - 녹색 채소 과다 등).

    영양제 섭취 타이밍: 식단의 지방 함량이 높으면 '지용성 영양제(오메가3, 비타민D 등)를 지금 드시면 흡수율이 좋다'는 팁을 추가.

    출력 형식: >    - is_safe: Boolean (주의 필요 여부)

        warning_message: String (위험 경고)

        action_guide: String (어떻게 먹으라는 조언)

        supplement_tip: String (영양제 타이밍 추천)

기존 분석기들의 MealAnalysis 결과물을 입력값으로 받아서 위 가이드를 생성하는 함수 check_safety(analysis_result, user_profile)를 작성해줘."
