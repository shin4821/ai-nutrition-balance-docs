현재 작성된 safety_guard.py의 로직을 고도화해줘. 사용자가 약물을 언제 복용하는지에 대한 시간 정보까지 고려해서 초정밀 케어 메시지를 생성하고 싶어.

주요 수정 사항:

1. 시간 정보 전달: check_safety 함수가 실행되는 현재 시간(예: 12:30)을 LLM에게 전달해줘. 
2. 사용자 프로필 구조 반영: user_profile.json의 medications가 단순 리스트가 아니라 name, time_of_day, note를 포함한 객체 리스트임을 반영해서 system_prompt를 수정해줘. 
3. 복합 분석 로직 추가: >    - 현재 식단의 '지방 함량'과 사용자가 아침에 먹은 '약물'의 상관관계 분석. 
   - 현재 시간과 가장 가까운 '영양제(예: 오메가3)' 복용 시간을 대조하여 최적의 타이밍 제안.
4. 페르소나 강화: 응답 멘트가 단순히 데이터 나열이 아니라, "지금 드시는 식사는 지방이 적절해 아침에 드신 약의 흡수율에 영향을 주지 않으면서, 저녁 영양제인 오메가3를 먹기 딱 좋은 타이밍입니다!"와 같이 친절하고 전문적인 문장으로 나오게 action_guide와 supplement_tip 규칙을 보강해줘."
