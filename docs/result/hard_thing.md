. 개발 중 난관 및 해결 방안

[문제점] 비정형 데이터 기반의 정밀 Safety Guard 구현

음식 사진과 사용자 프로필(질환, 약물)이라는 서로 다른 성격의 비정형 데이터를 결합하여, 실시간으로 정확한 ‘안전 경고’를 도출하는 로직 구현이 가장 까다로웠습니다. AI가 단순히 설명을 나열하는 것이 아니라, 사용자의 특정 질환에 맞는 수치적/성분적 위험도를 정밀하게 필터링해야 했습니다.

[해결 방안] Structured Output 및 Context Injection

1. 데이터 구조화 (Structured Output): OpenAI의 JSON Mode를 활용하여 식단 성분과 약물 정보를 엄격한 스키마(Schema)로 정의했습니다. 이를 통해 AI의 답변이 정해진 필드(안전 여부, 경고 메시지, 식사 가이드, 영양제 팁)로만 출력되도록 강제하여, 프론트엔드에서 일관된 UI로 표시하고 Hallucination을 방지했습니다.
2. 보안 기반 데이터 연동: 민감한 프로필 정보를 안전하게 전달하기 위해 Supabase Auth와 연동된 토큰 기반의 백엔드 요청 프로세스를 구축했습니다.
3. 다중 데이터 교차 검증: 식단 사진에서 추출한 영양소와 사용자의 기저 질환 정보를 백엔드에서 결합하여 전달하는 Context Injection 기법을 사용했습니다. 이를 통해 AI가 단순 식단 분석을 넘어 "현재 고혈압을 앓고 계시므로 이 음식의 나트륨 함량은 위험합니다"와 같은 초개인화된 안전 피드백을 내놓도록 최적화했습니다.
