결과물에 가장 큰 영향을 준 설계된 컨텍스트

[핵심 컨텍스트: "의료적 가이드라인이 반영된 구조적 시스템 프롬프트(System Prompt)"]

가장 큰 영향을 준 것은 단순한 질문이 아니라, AI에게 **'임상 영양사 및 약학적 조언자'**라는 전문적 페르소나를 부여하고, 분석 결과를 **엄격한 JSON 스키마(Structured Output)**로 출력하도록 설계한 컨텍스트입니다.

[설계의 핵심 포인트]

1. Role-Based Prompting: AI에게 단순한 챗봇이 아닌, 사용자의 신체 데이터와 질병 정보를 기반으로 위험 요소를 판단하는 **'보건 의료 전문가'**의 규칙을 주입했습니다. 이 규칙은 "몸에 좋다"는 일반론보다 "사용자의 질환에 해롭다"는 부정적 상호작용(Contraindication) 탐지를 우선순위에 두도록 설계되었습니다.

2. Data Structure Constraints (Schema): 비정형 이미지를 분석할 때 food_items, nutrients, safety_warnings 등 데이터 필드를 명확히 정의했습니다. 이 설계된 스키마 덕분에 AI의 답변이 매번 달라지는 현상을 방지하고, 서비스 UI에 안정적으로 데이터를 뿌려줄 수 있었습니다.

3. Cross-Reference Logic: 사용자의 프로필(질병, 약물) 데이터를 외부 컨텍스트로 주입하여, 식단 분석 시 이 데이터를 반드시 참조(Reference)하여 결론을 내리도록 강제하는 **'분석 규칙'**이 Safety Guard 기능의 정밀도를 결정지었습니다.
